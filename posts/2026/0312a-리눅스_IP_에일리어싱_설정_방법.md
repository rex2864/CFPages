## 리눅스 IP 에일리어싱 설정 방법

(gemini로 작성됨)

리눅스 서버를 운영하다 보면 하나의 물리적인 네트워크 인터페이스(NIC)에 여러 개의 IP 주소를 할당해야 할 때가 있죠.
예를 들어 한 서버에서 여러 개의 웹사이트를 운영하거나, 서비스 중단 없이 IP를 교체해야 하는 상황 말이죠.

오늘은 데비안(Debian)이나 우분투(Ubuntu) 계열에서 흔히 사용하는 **IP Aliasing(IP 에일리어싱)** 설정을 통해 하나의 인터페이스에 보조 IP를 추가하는 방법을 정리해 보겠습니다.


### IP Aliasing이란?

**IP Aliasing**은 하나의 물리적 네트워크 인터페이스(예: `eth0`)에 여러 개의 논리적 인터페이스(예: `eth0:0`, `eth0:1`)를 생성하여 각각 다른 IP를 부여하는 기술입니다.

#### 주요 활용 사례

* **가상 호스팅:** 하나의 서버에서 여러 SSL 인증서를 사용하거나 독립적인 IP가 필요한 여러 서비스를 구동할 때.
* **네트워크 마이그레이션:** 기존 IP를 유지하면서 새로운 IP를 추가하여 점진적으로 서비스를 이전할 때.
* **고가용성(HA):** 장애 조치(Failover)를 위해 가상 IP(VIP)를 설정할 때.


### 설정 방법 (`/etc/network/interfaces`)

리눅스 네트워크 설정 파일인 `/etc/network/interfaces`를 수정하여 영구적으로 IP를 할당할 수 있습니다.

#### 1. 설정 파일 수정

터미널에서 설정 파일을 엽니다.

```bash
sudo nano /etc/network/interfaces
```

#### 2. 네트워크 구성 예시

파일 하단에 아래와 같이 내용을 추가하거나 수정합니다.

```text
# 기본 네트워크 인터페이스 (Primary IP)
auto eth0
iface eth0 inet static
    address 192.168.0.2/24
    gateway 192.168.0.1

# 보조 네트워크 인터페이스 (Secondary IP)
auto eth0:0
iface eth0:0 inet static
    address 192.168.0.3/24
```

* **`eth0`**: 실제 물리적 인터페이스 이름입니다.
* **`eth0:0`**: 첫 번째 가상 인터페이스를 의미합니다. 추가가 더 필요하다면 `eth0:1`, `eth0:2` 식으로 늘려갈 수 있습니다.
* **`static`**: 고정 IP 방식을 사용함을 명시합니다.


### 설정 적용 및 확인

설정을 마쳤다면 네트워크 서비스를 재시작하거나 인터페이스를 다시 올립니다.

#### 설정 적용

```bash
sudo ifup eth0:0
```

*(또는 전체 네트워크 서비스 재시작: `sudo systemctl restart networking`)*

#### IP 할당 확인

설정이 제대로 되었는지 확인하려면 아래 명령어를 입력하세요.

```bash
ip addr show eth0
```

출력 결과에서 `192.168.0.2`와 `192.168.0.3`이 모두 `eth0`에 할당되어 있다면 성공입니다!


### 주의사항

1. **넷마스크(Netmask):** 위 예시처럼 `/24` 형식을 사용하거나 `netmask 255.255.255.0`을 명시해야 합니다.
2. **게이트웨이:** 일반적으로 게이트웨이는 물리 인터페이스(`eth0`)에 하나만 설정하면 됩니다. 보조 인터페이스에 중복 설정할 경우 라우팅 경로에 혼선이 생길 수 있습니다.
